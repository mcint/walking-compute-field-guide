# Naming Notes

The repo is `walking-compute-field-guide`.

The project uses both “walking compute” and “walkable compute” language:

- **walking compute**: compute while standing, pacing, walking, or moving through a space.
- **walkable compute**: compute designed to be socially and ergonomically viable away from a desk.

## Names considered

| Name | Vibe | Notes |
|---|---|---|
| `walking-compute-field-guide` | durable | Current repo name. Broad enough for buyer guide, field reports, and norms. |
| `walkable-compute-field-guide` | durable | Slightly more ergonomic/social framing. |
| `walking-compute-buyers-guide` | narrow | Good first task, too small long-term. |
| `awesome-walking-compute` | curated list | Better later, once there is enough material to curate. |
| `walc` | acronym bait | Walkable Augmented Linux Compute. Cute, opaque. |
| `nomad` | identity | Noisebridge Open Mobile Augmented Desktop. Good backronym, less searchable. |
| `gazebo` | social pun | Gaze-aware zero-desk ergonomic belt outfit. Extremely Noisebridge, suspiciously overfit. |
| `sidewalk-shell` | software-forward | Good if the project becomes terminal/workflow heavy. |
| `hip-compute` | input/mounting-forward | Accurate and goofy. |

## Scope implied by current name

A field guide can include:

- Buyer guide.
- Compatibility matrix.
- Known-good rigs.
- Failure reports.
- Social norms.
- Projector/social-cast experiments.
- MediaWiki pages.
- Issue and PR workflows.
- Later: an `awesome-*` curated page.
